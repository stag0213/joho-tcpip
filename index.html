TCP/IP 4階層モデルとは

TCP/IP 4階層モデルは、インターネット通信を実現するためのプロトコル体系です。データ通信を4つの階層に分けて、それぞれが独立した役割を持ちながら連携することで、世界中のコンピュータ間の通信を可能にしています。

第4層

アプリケーション層

ユーザーが直接利用するアプリケーションのための層

HTTP HTTPS FTP SMTP DNS
第3層

トランスポート層

信頼性のあるデータ転送を管理する層

TCP UDP
第2層

インターネット層

データの経路選択とアドレス管理を行う層

IP ICMP ARP
第1層

ネットワークインターフェース層

物理的なネットワーク接続を担当する層

Ethernet Wi-Fi PPP

各階層の詳細

アプリケーション層 (Application Layer)

ユーザーアプリケーションと直接やり取りする最上位層

主な役割

  • ユーザーアプリケーションへのネットワークサービス提供
  • データの形式変換と暗号化
  • アプリケーション間の通信規約の定義

主要プロトコル

HTTP/HTTPS

Webページの表示（ポート80/443）

FTP

ファイル転送（ポート20/21）

SMTP/POP3/IMAP

メール送受信（ポート25/110/143）

DNS

ドメイン名解決（ポート53）

トランスポート層 (Transport Layer)

エンドツーエンドの通信制御を担当

主な役割

  • データの分割と再構成
  • 信頼性のある通信の保証
  • ポート番号による通信先の識別
  • フロー制御と輻輳制御

主要プロトコル

TCP (Transmission Control Protocol)

✓ 信頼性が高い（再送機能あり）
✓ コネクション型通信
✓ 順序保証、エラー検出
用途: Web、メール、ファイル転送

UDP (User Datagram Protocol)

✓ 高速だが信頼性は低い
✓ コネクションレス型
✓ オーバーヘッドが少ない
用途: 動画配信、DNS、オンラインゲーム

インターネット層 (Internet Layer)

異なるネットワーク間でのデータ転送を実現

主な役割

  • IPアドレスによる宛先の特定
  • ルーティング（経路選択）
  • パケットの分割と再構成
  • ネットワーク間の中継

主要プロトコル

IP (Internet Protocol)

IPv4: 32ビットアドレス (例: 192.168.1.1)
IPv6: 128ビットアドレス (例: 2001:db8::1)

ICMP (Internet Control Message Protocol)

エラー通知、診断ツール（ping, traceroute）

ARP (Address Resolution Protocol)

IPアドレスとMACアドレスの対応付け

ネットワークインターフェース層 (Network Interface Layer)

物理的なネットワーク接続とデータ伝送

主な役割

  • 物理メディアへのデータ送信
  • MACアドレスによる機器識別
  • フレーム化とエラー検出
  • 同一ネットワーク内の通信

主要技術・プロトコル

Ethernet

有線LANの標準規格
MACアドレス（48ビット）を使用

Wi-Fi (IEEE 802.11)

無線LANの標準規格
Wi-Fi 6 (802.11ax) など

PPP (Point-to-Point Protocol)

2点間通信プロトコル

データの流れ - カプセル化とデカプセル化

処理の詳細

ボタンをクリックして、データの流れを確認してください。

各ヘッダーに含まれる情報

Ethernetヘッダ

  • 送信元MACアドレス
  • 宛先MACアドレス
  • タイプ（IPv4/IPv6）

IPヘッダ

  • 送信元IPアドレス
  • 宛先IPアドレス
  • TTL（生存時間）
  • プロトコル番号

TCPヘッダ

  • 送信元ポート番号
  • 宛先ポート番号
  • シーケンス番号
  • 確認応答番号
  • フラグ（SYN, ACK, FINなど）

FCS (トレーラー)

  • フレームチェックシーケンス
  • エラー検出用のCRC値

実例で理解する

例1: Webブラウジング（HTTPSでWebページを表示）

1

ユーザーの操作

ブラウザのアドレスバーに「https://www.example.com」と入力してEnterキーを押す

2

アプリケーション層

DNS: ドメイン名「www.example.com」をIPアドレスに変換
HTTPS: HTTPリクエストを作成し、TLS/SSLで暗号化
GET / HTTP/1.1
Host: www.example.com

3

トランスポート層

TCP: ポート443（HTTPS）への接続を確立
3ウェイハンドシェイク（SYN → SYN-ACK → ACK）
データを分割してセグメント化

4

インターネット層

IP: 送信元と宛先のIPアドレスを設定
ルーティングテーブルを参照して最適な経路を決定
パケット化

5

ネットワークインターフェース層

Ethernet/Wi-Fi: MACアドレスを付加してフレーム化
物理的な信号に変換してネットワークに送出
ルーターやスイッチを経由してサーバーへ到達

6

サーバーからの応答

サーバーが同じプロセスを逆順で処理（デカプセル化）
HTTPSレスポンスを生成し、同様の手順で送信（カプセル化）
ブラウザがHTMLを受信・解析してWebページを表示

例2: メール送信（SMTP）

1

ユーザーの操作

メールソフトでメールを作成し、「送信」ボタンをクリック

2

アプリケーション層

SMTP: メール送信プロトコル（ポート25/587）
メールヘッダーと本文を規定のフォーマットで構成
From: sender@example.com
To: receiver@example.com
Subject: こんにちは

3

トランスポート層

TCP: ポート587への接続（SMTP over TLS）
信頼性のある配送を保証
エラー時は再送を実施

4

インターネット層〜ネットワークインターフェース層

Webブラウジングと同様のプロセスでメールサーバーへ配送
受信者のメールサーバーまで複数のメールサーバーを中継

5

受信者のメール取得

POP3/IMAP: 受信者がメールサーバーからメールを取得
POP3（ポート110）: メールをダウンロード
IMAP（ポート143）: サーバー上でメール管理

↓

例3: オンラインゲーム（UDP）

なぜUDPを使用？

  • 低遅延が重要: リアルタイム性が求められる
  • 多少のパケットロスは許容: 1フレーム分のデータが失われても次のフレームで更新される
  • 高速通信: TCPのような接続確立や再送がないため高速

プロトコルスタック

ゲームプロトコル
↓
UDP
↓
IP
↓
Ethernet / Wi-Fi

まとめ

階層化の利点

各層が独立して機能するため、一部を変更しても他の層に影響しない。標準化により異なるベンダーの機器が相互運用可能。

役割分担

各階層が明確な役割を持ち、上位層はより抽象的なサービスを、下位層はより具体的な処理を担当。

カプセル化

各階層でヘッダーを追加することで、その層に必要な情報を付加。受信側では逆順で処理（デカプセル化）。

インターネットの基盤

TCP/IP 4階層モデルは、インターネット通信の基盤となる重要な概念。理解することでネットワークトラブルの解決に役立つ。
